📋 Lista de Tareas Priorizadas para RyDit Engine

1. Sistema de módulos avanzado + CI/CD (v0.2.0)

· Implementar caché de módulos para evitar recargas innecesarias.
· Detectar y manejar dependencias circulares entre módulos.
· Configurar GitHub Actions para que corra cargo test en cada push y PR.
· Por qué primero: Es la base para que el ecosistema de scripts pueda crecer sin problemas de rendimiento ni errores de importación. Además, el CI/CD garantiza que los tests (110 actuales) sigan pasando con cada cambio.

2. Manejo básico de assets (texturas, sonidos)

· Añadir funciones en el lenguaje para cargar y dibujar sprites (assets::load_texture, draw::texture).
· Soportar formatos comunes (PNG, WAV, etc.) usando raylib.
· Por qué segundo: Los juegos necesitan gráficos más allá de formas primitivas. Es el siguiente paso natural después de tener el game loop estable.

3. Mejora de mensajes de error en tiempo de ejecución

· Mostrar línea, columna y causa del error cuando un script RyDit falle (ej. división por cero, acceso fuera de rango).
· Por qué tercero: Una buena experiencia de depuración es clave para que nuevos usuarios puedan aprender y crear sin frustrarse.

4. Bot de ayuda en shell (rybot.sh)

· Crear un script bash con menú interactivo que muestre:
  · Sintaxis básica del lenguaje.
  · Lista de comandos gráficos (draw.*).
  · Ejecutar demos (shapes, snake, etc.) con un solo comando.
  · Enlaces a documentación.
· Por qué cuarto: Es una solución inmediata, ligera y que no requiere infraestructura externa. Ayuda a los usuarios desde el primer momento en Termux.

5. Bot de Telegram (prototipo)

· Implementar un bot simple (en Rust o Python) que responda con ayuda sobre comandos, ejemplos y documentación.
· Por qué quinto: Extiende la accesibilidad fuera de la terminal, pero requiere alojamiento y mantenimiento. Puede lanzarse después de tener el bot en shell.
